# 17 - 好友系统重构设计文档

> 编写日期：2026-07-01 | 版本：v1.0

## 一、问题背景

1. **前端API路径不匹配**：friend.ts路径与后端路由不一致，所有好友操作404
2. **缺少"已发送邀请"状态**：发送请求后搜索结果中该用户直接消失，无视觉反馈
3. **缺少"我发出的请求"列表**：只展示收到的好友请求
4. **拒绝后无法重新发送**：REJECTED状态永久保留

## 二、重构目标

1. 清理现有好友数据，从干净状态开始
2. 发送请求后搜索结果中显示"已发送邀请"
3. 好友关系必须经对方同意才建立
4. 新增"我发出的请求"列表
5. 拒绝后允许重新发送请求

## 三、数据库Schema

### 3.1 无结构变更

Friendship表已满足需求，无需修改。FriendStatus枚举保持 PENDING/ACCEPTED/REJECTED。

### 3.2 数据清理方案

```sql
DELETE FROM friendships;
```

或通过 `npx prisma db push --force-reset && npx prisma db seed` 全量重置。

## 四、后端API接口设计

### 4.1 发送好友请求

```
POST /api/v1/friend/request
Body: { "targetUserId": "xxx" }
Response 201: { code:0, message:"好友请求已发送", data: friendship }
```

逻辑变更：REJECTED记录→删除旧记录重新创建PENDING

### 4.2 同意好友请求

```
POST /api/v1/friend/accept/:friendshipId
Response 200: { code:0, message:"已添加好友" }
```

### 4.3 拒绝好友请求

```
POST /api/v1/friend/reject/:friendshipId
Response 200: { code:0, message:"已拒绝" }
```

### 4.4 获取好友列表

```
GET /api/v1/friend/list
Response 200: { code:0, data: Friend[] }
```

### 4.5 获取收到的待处理请求（路径变更）

```
GET /api/v1/friend/requests/received
Response 200: { code:0, data: FriendRequest[] }
```

### 4.6 获取发出的待处理请求（新增）

```
GET /api/v1/friend/requests/sent
Response 200: { code:0, data: SentRequest[] }
```

### 4.7 搜索用户（增强返回值）

```
GET /api/v1/user/search?keyword=xxx
```

每个搜索结果新增 `friendStatus` 字段：

| 值 | 含义 |
|---|---|
| NONE | 无好友关系 |
| PENDING_SENT | 我已发送请求，等待对方同意 |
| PENDING_RECEIVED | 对方已向我发送请求 |
| ACCEPTED | 已经是好友 |
| REJECTED | 我之前被拒绝（可重新发送） |

## 五、后端Service变更

### 5.1 sendFriendRequest

REJECTED状态时删除旧记录重新创建：
```
existing.status === 'REJECTED' → delete old → create new PENDING
```

### 5.2 新增 getSentRequests

查询 requesterId=userId AND status=PENDING 的记录，include addressee信息。

### 5.3 searchUsers增强

搜索后批量查询当前用户与搜索结果用户的好友关系，为每个结果附加friendStatus。

## 六、前端API层变更（friend.ts）

```typescript
sendFriendRequestApi(targetUserId) → POST /friend/request
acceptFriendRequestApi(id)         → POST /friend/accept/{id}
rejectFriendRequestApi(id)         → POST /friend/reject/{id}
getFriendListApi()                 → GET /friend/list
getReceivedRequestsApi()           → GET /friend/requests/received  // 重命名
getSentRequestsApi()               → GET /friend/requests/sent      // 新增
```

## 七、前端Store变更（chat.ts）

### 7.1 新增状态

- `sentRequests: ref<SentRequest[]>([])` — 我发出的待处理请求

### 7.2 搜索结果类型增强

SearchResult新增 `friendStatus` 字段

### 7.3 sendFriendRequest变更

发送成功后更新搜索结果中该用户的friendStatus为PENDING_SENT，并刷新sentRequests。

## 八、前端UI交互

### 8.1 搜索结果按钮状态

| friendStatus | 按钮显示 | 可点击 |
|---|---|---|
| NONE | 添加好友 | 是 |
| PENDING_SENT | 已发送邀请 | 否（灰色） |
| PENDING_RECEIVED | 同意请求 | 是（跳转到请求列表） |
| ACCEPTED | 已是好友 | 否 |
| REJECTED | 重新发送 | 是 |

### 8.2 好友面板分区

```
好友面板
├── 搜索栏（输入UID/用户名搜索）
├── 搜索结果（带friendStatus状态按钮）
├── 收到的请求（待处理，显示同意/拒绝按钮）
├── 发出的请求（新增，显示"等待对方同意"）
└── 我的好友（点击发起私聊）
```
