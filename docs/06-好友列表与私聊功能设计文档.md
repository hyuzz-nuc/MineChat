# 06 - 好友列表与私聊功能设计文档

> 版本: v1.0 | 日期: 2026-06-29 | 状态: 设计中

## 1. 问题描述

当前 ChatView 导航栏有"联系人👥"图标，但**没有绑定任何功能**。用户点击后无反应。需要实现好友列表面板和从好友发起私聊的完整流程。

## 2. 功能需求

### 2.1 好友列表面板

- 点击导航栏"👥"图标，侧边栏切换到**好友列表视图**
- 好友列表显示：头像、昵称、在线状态
- 支持搜索好友
- 显示待处理的好友请求（红点徽标）

### 2.2 发起私聊

- 点击好友 → 检查是否已有私聊房间
  - 有 → 直接切换到该房间
  - 无 → 调用 `createDirectRoom` API 创建，然后切换
- 创建后自动出现在会话列表顶部

### 2.3 好友请求

- 在好友列表顶部显示"好友请求"入口
- 显示待处理的请求数量
- 点击进入请求列表，可接受/拒绝

## 4. API对接

### 4.1 已有API（可直接使用）

| API | 方法 | 说明 |
|-----|------|------|
| `/api/v1/friends` | GET | 获取好友列表 |
| `/api/v1/friends/pending` | GET | 获取待处理请求 |
| `/api/v1/friends/:id/accept` | PATCH | 接受好友请求 |
| `/api/v1/friends/:id/reject` | PATCH | 拒绝好友请求 |
| `/api/v1/rooms/direct` | POST | 创建私聊房间 |
| `/api/v1/user/search?q=` | GET | 搜索用户 |

### 4.2 交互流程

```
用户点击好友
    ↓
调用 createDirectRoomApi(friendId)
    ↓
返回 { roomId }
    ↓
调用 selectConversation(roomId)
    ↓
切换到聊天视图
```
