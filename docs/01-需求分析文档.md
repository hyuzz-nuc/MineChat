## 一、项目概述

### 1.1 项目定位

MineChat 是一款 Web 端实时聊天软件，追求高级感与沉浸感视觉体验。项目面向注重审美与交互品质的用户群体，提供流畅、安全、精致的即时通讯服务。

### 1.2 核心价值主张

| 维度 | 描述 |
|------|------|
| **视觉沉浸** | 暗色系 + 毛玻璃 + 微光效，打造影院级聊天体验 |
| **交互流畅** | 实时消息推送、丝滑动画过渡、零卡顿操作 |
| **安全可靠** | 端到端加密、数据安全存储、隐私保护 |
| **简洁高效** | 功能精炼不臃肿，核心体验做到极致 |

### 1.3 目标用户

- 追求高品质视觉体验的年轻用户（18-35岁）
- 对隐私安全有较高要求的用户
- 喜欢暗色主题、沉浸式交互的科技爱好者
- 小团队/社群的私密沟通需求

## 三、非功能需求

### 3.1 性能要求

| 指标 | 目标值 |
|------|--------|
| 消息延迟 | < 200ms（同区域） |
| 并发连接 | 支持 1000+ WebSocket 连接 |
| 首屏加载 | < 2s（正常网络） |
| 动画帧率 | ≥ 60fps |
| 内存占用 | 前端 < 150MB |

### 3.2 安全要求

| 要求 | 描述 |
|------|------|
| 传输安全 | 全站 HTTPS/WSS |
| 密码安全 | bcrypt 加密存储，salt rounds ≥ 12 |
| 鉴权机制 | JWT + Refresh Token 双令牌 |
| XSS防护 | 消息内容严格转义 |
| SQL注入 | 参数化查询 |
| 文件上传 | 类型校验、大小限制 |
| 频率限制 | API 请求频率限制，防暴力破解 |

### 3.3 可用性要求

| 要求 | 描述 |
|------|------|
| 浏览器兼容 | Chrome 90+、Firefox 88+、Safari 14+、Edge 90+ |
| 响应式 | 适配桌面端（1280px+）和平板端（768px+） |
| 无障碍 | 基础键盘导航、ARIA 标签 |
| 离线提示 | 网络断开时明确提示 |

### 3.4 可扩展性

| 要求 | 描述 |
|------|------|
| 水平扩展 | 后端无状态设计，支持多实例部署 |
| 消息队列 | 预留消息队列接入点 |
| 存储扩展 | 支持对象存储切换（本地/云存储） |

### 4.2 关键用例详述

#### UC-01：用户注册

- **参与者**：未注册用户
- **前置条件**：用户访问 MineChat 注册页面
- **主流程**：
  1. 用户输入用户名、邮箱、密码
  2. 系统校验用户名唯一性和邮箱格式
  3. 系统加密密码并创建账户
  4. 系统返回 JWT Token，自动登录
  5. 跳转至主界面
- **备选流程**：
  - 用户名已存在 → 提示"用户名已被占用"
  - 邮箱已注册 → 提示"邮箱已注册，是否登录"

#### UC-02：发送消息

- **参与者**：已登录用户
- **前置条件**：用户已打开聊天窗口
- **主流程**：
  1. 用户在输入框输入消息内容
  2. 对方看到"正在输入..."状态
  3. 用户点击发送或按 Enter
  4. 消息通过 WebSocket 发送至服务端
  5. 服务端广播消息给接收方
  6. 发送方显示"已发送"→"已送达"状态
  7. 接收方阅读后，发送方显示"已读"
- **备选流程**：
  - 网络断开 → 消息进入待发送队列，恢复后自动重发
  - 接收方离线 → 消息存储为离线消息，上线后推送

#### UC-03：创建群组

- **参与者**：已登录用户
- **前置条件**：用户至少有1个好友
- **主流程**：
  1. 用户点击"创建群组"按钮
  2. 输入群名称、选择群头像
  3. 从好友列表中选择成员（至少1人）
  4. 系统创建群组并生成群ID
  5. 系统向被邀请成员发送群邀请通知
  6. 跳转至群聊界面
- **备选流程**：
  - 群名称为空 → 提示"请输入群名称"
  - 未选择成员 → 提示"请至少选择1位成员"

## 六、约束与假设

### 6.1 约束

1. 当前阶段仅开发 Web 端，不考虑移动端 App
2. 初期用户量预估 < 5000，架构需支持后续扩展
3. 服务器资源有限，需合理控制成本
4. 不依赖第三方IM服务，全自研

### 6.2 假设

1. 用户网络环境良好，WebSocket 连接稳定
2. 用户使用现代浏览器，支持 WebSocket/ES2020+
3. 图片/文件存储初期使用服务器本地磁盘
4. 单台服务器可满足初期并发需求

## 七、验收标准

### 7.1 P0 功能验收

| 序号 | 验收项 | 验收标准 |
|------|--------|----------|
| 1 | 用户注册 | 能成功注册并自动登录，信息持久化 |
| 2 | 用户登录 | 正确凭证登录成功，错误凭证拒绝 |
| 3 | 私聊消息 | 消息实时送达，延迟 < 500ms |
| 4 | 群聊消息 | 群消息所有成员实时收到 |
| 5 | 消息状态 | 已发送/已送达/已读状态正确显示 |
| 6 | 离线消息 | 上线后自动收到离线期间消息 |
| 7 | 好友系统 | 能搜索/添加/删除好友 |
| 8 | 视觉品质 | 暗色毛玻璃主题，动画流畅无卡顿 |

### 7.2 非功能验收

| 序号 | 验收项 | 验收标准 |
|------|--------|----------|
| 1 | 性能 | 首屏加载 < 2s，消息延迟 < 500ms |
| 2 | 安全 | HTTPS全站加密，密码不明文存储 |
| 3 | 兼容 | Chrome/Firefox/Edge/Safari 最新版正常运行 |
| 4 | 稳定 | 连续运行24h无内存泄漏、无崩溃 |

## 八、术语表

| 术语 | 定义 |
|------|------|
| WebSocket | 全双工通信协议，用于实时消息推送 |
| JWT | JSON Web Token，无状态鉴权令牌 |
| 毛玻璃效果 | backdrop-filter: blur() 实现的磨砂玻璃视觉 |
| 乐观更新 | 消息发送后立即在UI显示，不等服务端确认 |
| 离线队列 | 用户离线期间消息暂存，上线后批量推送 |
