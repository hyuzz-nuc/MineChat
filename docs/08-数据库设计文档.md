## 二、表结构详解

### 2.1 users — 用户表

| 字段 | 类型 | 约束 | 描述 |
|------|------|------|------|
| id | VARCHAR(30) | PK, cuid | 用户唯一ID |
| username | VARCHAR(32) | UQ, NOT NULL | 用户名（登录用） |
| email | VARCHAR(255) | UQ, NOT NULL | 邮箱（注册用） |
| password_hash | VARCHAR(255) | NOT NULL | bcrypt加密密码 |
| nickname | VARCHAR(32) | NOT NULL | 显示昵称 |
| avatar | VARCHAR(512) | NULL | 头像URL |
| bio | VARCHAR(200) | NULL | 个性签名 |
| status | ENUM | NOT NULL, DEFAULT OFFLINE | 在线状态(ONLINE/OFFLINE/BUSY) |
| last_seen_at | TIMESTAMP | NULL | 最后在线时间 |
| created_at | TIMESTAMP | NOT NULL, DEFAULT NOW() | 创建时间 |
| updated_at | TIMESTAMP | NOT NULL, AUTO UPDATE | 更新时间 |

### 2.2 rooms — 房间表（私聊/群聊统一）

| 字段 | 类型 | 约束 | 描述 |
|------|------|------|------|
| id | VARCHAR(30) | PK, cuid | 房间唯一ID |
| type | ENUM | NOT NULL, DEFAULT DIRECT | 房间类型(DIRECT/GROUP) |
| name | VARCHAR(64) | NULL | 房间名（群聊用） |
| avatar | VARCHAR(512) | NULL | 房间头像（群聊用） |
| last_msg_at | TIMESTAMP | NOT NULL, DEFAULT NOW() | 最后消息时间（排序用） |
| created_at | TIMESTAMP | NOT NULL | 创建时间 |
| updated_at | TIMESTAMP | NOT NULL | 更新时间 |

**设计说明**：私聊和群聊统一在rooms表中，通过type字段区分。私聊room的name为null，由前端根据成员信息动态显示。

### 2.3 room_members — 房间成员表

| 字段 | 类型 | 约束 | 描述 |
|------|------|------|------|
| id | VARCHAR(30) | PK, cuid | 成员记录ID |
| room_id | VARCHAR(30) | FK, NOT NULL | 所属房间 |
| user_id | VARCHAR(30) | FK, NOT NULL | 成员用户 |
| role | ENUM | NOT NULL, DEFAULT MEMBER | 角色(OWNER/ADMIN/MEMBER) |
| last_read | TIMESTAMP | NULL | 最后阅读时间（计算未读数） |
| joined_at | TIMESTAMP | NOT NULL | 加入时间 |

**联合唯一约束**：`(room_id, user_id)`

**设计说明**：`last_read`字段用于计算未读消息数。查询未读数 = 该房间中 `created_at > last_read` 的消息数。

### 2.4 messages — 消息表

| 字段 | 类型 | 约束 | 描述 |
|------|------|------|------|
| id | VARCHAR(30) | PK, cuid | 消息唯一ID |
| room_id | VARCHAR(30) | FK, NOT NULL | 所属房间 |
| sender_id | VARCHAR(30) | FK, NOT NULL | 发送者 |
| type | ENUM | NOT NULL, DEFAULT TEXT | 消息类型(TEXT/IMAGE/FILE/SYSTEM) |
| content | TEXT | NOT NULL | 消息内容 |
| status | ENUM | NOT NULL, DEFAULT SENT | 状态(SENT/DELIVERED/READ) |
| reply_to | VARCHAR(30) | NULL | 引用消息ID |
| deleted_at | TIMESTAMP | NULL | 软删除时间 |
| created_at | TIMESTAMP | NOT NULL | 发送时间 |

**索引**：
- `(room_id, created_at)` — 查询房间消息列表（最频繁的查询）
- `(sender_id)` — 查询用户发送的消息

### 2.5 friendships — 好友关系表

| 字段 | 类型 | 约束 | 描述 |
|------|------|------|------|
| id | VARCHAR(30) | PK, cuid | 关系ID |
| requester_id | VARCHAR(30) | FK, NOT NULL | 申请人 |
| addressee_id | VARCHAR(30) | FK, NOT NULL | 被申请人 |
| status | ENUM | NOT NULL, DEFAULT PENDING | 状态(PENDING/ACCEPTED/REJECTED) |
| created_at | TIMESTAMP | NOT NULL | 申请时间 |
| updated_at | TIMESTAMP | NOT NULL | 更新时间 |

**联合唯一约束**：`(requester_id, addressee_id)`

### 2.6 groups — 群组扩展信息

| 字段 | 类型 | 约束 | 描述 |
|------|------|------|------|
| id | VARCHAR(30) | PK, cuid | 群组ID |
| room_id | VARCHAR(30) | FK, UQ, NOT NULL | 对应房间 |
| owner_id | VARCHAR(30) | FK, NOT NULL | 群主 |
| description | VARCHAR(200) | NULL | 群描述 |
| announcement | VARCHAR(500) | NULL | 群公告 |

### 2.7 file_uploads — 文件上传记录

| 字段 | 类型 | 约束 | 描述 |
|------|------|------|------|
| id | VARCHAR(30) | PK, cuid | 文件ID |
| user_id | VARCHAR(30) | FK, NOT NULL | 上传者 |
| file_name | VARCHAR(255) | NOT NULL | 原始文件名 |
| file_path | VARCHAR(512) | NOT NULL | 存储路径 |
| file_size | INT | NOT NULL | 文件大小(bytes) |
| mime_type | VARCHAR(128) | NOT NULL | MIME类型 |
| created_at | TIMESTAMP | NOT NULL | 上传时间 |

## 三、关键查询与索引设计

### 3.1 高频查询

| 查询场景 | SQL思路 | 索引支持 |
|----------|---------|----------|
| 获取会话列表 | 查用户所在rooms，按last_msg_at排序 | room_members.user_id |
| 获取房间消息 | 查messages WHERE room_id ORDER BY created_at | (room_id, created_at) |
| 计算未读数 | 查messages WHERE room_id AND created_at > last_read | (room_id, created_at) |
| 搜索用户 | WHERE username LIKE '%keyword%' | username唯一索引 |
| 好友列表 | friendships WHERE (requester_id OR addressee_id) AND status=ACCEPTED | 联合查询 |

### 3.2 索引清单

| 表 | 索引 | 类型 | 用途 |
|----|------|------|------|
| users | username | UNIQUE | 用户名查找 |
| users | email | UNIQUE | 邮箱查找 |
| messages | (room_id, created_at) | COMPOSITE | 房间消息列表 |
| messages | sender_id | SINGLE | 用户消息查询 |
| friendships | (requester_id, addressee_id) | UNIQUE | 唯一好友关系 |
| room_members | (room_id, user_id) | UNIQUE | 唯一成员关系 |

## 五、数据库初始化指南

```bash
# 1. 安装 PostgreSQL 15+，创建数据库
createdb minechat

# 2. 配置 .env
cp server/.env.example server/.env
# 编辑 DATABASE_URL

# 3. 生成 Prisma Client
cd server && pnpm db:generate

# 4. 推送 Schema 到数据库（开发阶段用 db:push）
pnpm db:push

# 5. 后续正式使用迁移
pnpm db:migrate
```
