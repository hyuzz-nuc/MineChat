# 14-开机动画与登录页粒子汇聚设计文档

## 1. 设计目标

参照 MineRadio 开机动画效果，为 MineChat 设计一套沉浸式开机动画：
- 粒子从屏幕四周汇聚至中心，形成品牌文字
- 光束线条扫描屏幕
- 分形碎片从中心向外扩散
- 品牌文字"MineChat"以遮罩展开动画出现
- 5秒后自动过渡到登录界面，或点击/按键提前跳转

## 2. MineRadio 开机动画核心分析

| 元素 | 实现方式 | 效果描述 |
|------|----------|----------|
| 背景 | WebGL shader + CSS渐变 | 深色底色带扫描线和呼吸光效 |
| 尘埃粒子(dust) | Canvas 2D, 84个 | 微小白色圆点缓慢飘动，带呼吸透明度 |
| 光束(streaks) | Canvas 2D, 水平线条 | 带渐变色的水平光束滑过屏幕，有辉光 |
| 碎片(shards) | Canvas 2D, 平行四边形 | 从中心扩散的彩色碎片，带阴影辉光 |
| 信号线 | Canvas 2D, 中央水平线 | 从窄到宽的渐变线条，带脉冲光点 |
| 波形线 | Canvas 2D, 正弦波 | 模拟音波的多频叠加线条 |
| 闪光 | Canvas 2D, 矩形渐变 | 短暂全屏水平闪光 |
| 品牌文字 | CSS animation, clip-path | "Mine"从遮罩展开 → "radio"渐变文字出现 |
| 退出过渡 | CSS transform + opacity | scale(1.018) + fadeOut, 1.18s |

## 3. MineChat 开机动画设计

### 3.1 时间轴（总计 5秒）

```
0.0s ─ 0.7s   : 黑屏→深色背景渐入，尘埃粒子开始飘动
0.7s ─ 2.5s   : 光束线条扫描，碎片从中心扩散
2.0s ─ 3.5s   : 信号线展开，波形线出现
2.5s ─ 4.0s   : "Mine"文字遮罩展开
3.0s ─ 4.2s   : "Chat"文字渐变出现
3.8s ─ 4.5s   : 闪光脉冲
4.5s ─ 5.0s   : "点击进入"提示出现
5.0s+          : 用户交互 → 退出过渡 → 登录界面
```

### 3.2 粒子系统参数

| 类型 | 数量 | 大小 | 速度 | 色彩 | 透明度 |
|------|------|------|------|------|--------|
| 尘埃(dust) | 60-80 | 0.5-2px | 0.15px/frame | rgba(255,255,255) | 0.2-0.6 呼吸 |
| 光束(streak) | 8-12 | width 0.75-2.8px | 0.3-0.7px/frame | 渐变(#FF5367→#00F5D4→#F4D28A) | 带辉光 |
| 碎片(shard) | 12-18 | 18-86px | 从中心扩散 | rgba(244,210,138)/rgba(122,215,194)/rgba(255,83,103) | 带阴影 |
| 信号线 | 1 | 1.4-3.6px | 渐入渐出 | 渐变(#00F5D4→#F4D28A) | 带辉光shadowBlur |
| 波形线 | 1 | 1px | 正弦8.2Hz | rgba(244,210,138) | 0.22 |

### 3.3 品牌文字动画

```
MineChat 品牌文字分为 "Mine" + "Chat" 两段：

"Mine": 
  0-2.5s: opacity:0, clip-path:inset(48% 0 49% 0), skewX(-10deg)
  2.5-3.2s: 逐渐展开clip-path, skewX归零
  3.2s+: 完全展开, opacity:1

"Chat":
  0-3.0s: opacity:0, clip-path:inset(52% 0 44% 0)
  3.0-3.8s: 渐变文字出现, 背景色渐变(#00F5D4→#F4D28A→白色)
  3.8s+: 完全展开
```

### 3.4 色彩体系

- 背景基色: `#010304` (近乎纯黑，比MineChat主界面更深)
- 扫描线: `rgba(255,255,255,0.035)`
- 主accent: `#00F5D4` (MineChat绿)
- warm accent: `#F4D28A` (暖金，呼应MineRadio)
- hot accent: `#FF5367` (红粉，对比色)

### 3.5 技术实现

**方案选择：纯 Canvas 2D + CSS Animation**

- 不用 WebGL shader（MineChat 聊天软件不需要3D场景，减少复杂度）
- Canvas 2D 实现粒子、光束、碎片、波形
- CSS animation 实现文字遮罩展开
- 退出过渡用 CSS transition

**组件结构**：
```
App.vue
  ├── SplashOverlay.vue  (开机动画层，z-index:1000)
  │     ├── <canvas>      (粒子/光束/碎片/波形)
  │     ├── .splash-noise  (SVG噪声纹理)
  │     └ .splash-content  (品牌文字+提示)
  │
  ├── ParticleBackground.vue (聊天页背景粒子)
  ├── router-view           (LoginView / ChatView)
```

**状态管理**：
- `appState.splashVisible` — 控制开机动画显示
- 登录成功后自动隐藏开机动画
- 5秒后或用户点击/按键触发 `dismissSplash()`

## 4. 登录页层次设计

开机动画退出后，登录页需要有层次感：

```
登录页结构（z-index分层）：
z:0   — 深色背景 (#08090B)
z:1   — 粒子背景（继承ParticleBackground，但粒子更密集）
z:10  — 磨砂玻璃登录卡片（blur(34px), 居中）
z:20  — 品牌标识区（MineChat logo + 标语）
```

登录卡片设计：
- 磨砂玻璃效果 (backdrop-filter: blur(34px))
- 圆角16px
- 边框: 1px solid rgba(0,245,212,.15)
- 内部: 用户名/密码输入 + 登录按钮
- 底部: "还没有账号？注册"

## 5. 实现步骤

1. 创建 `splashEngine.ts` — 开机动画粒子引擎（dust/streak/shard/signal/wave）
2. 创建 `SplashOverlay.vue` — 开机动画组件
3. 重构 `LoginView.vue` — 添加粒子背景和磨砂玻璃层次
4. 在 `App.vue` 中集成开机动画
5. 添加 `appState` store 或在 App.vue 管理 splash 状态
