# 16-登录多方式与高级感背景设计文档

## 1. 需求概述

### 1.1 登录多方式
当前登录仅支持「用户名/邮箱 + 密码」，需新增：
- **手机号验证码登录**：输入手机号 → 发送验证码 → 输入验证码 → 登录
- **邮箱验证码登录**：输入邮箱 → 发送验证码 → 输入验证码 → 登录
- 保留原有「账号密码登录」

### 1.2 高级感背景
当前登录页背景仅有微弱的径向渐变和星系旋涡CSS层，视觉单薄。需：
- 复用 `ParticleBackground` 组件（含流星+星尘+连线+脉冲）
- 增强星系旋涡效果（多层叠加）
- 添加深空微光网格纹理

## 2. 后端设计

### 2.1 新增API接口

#### 发送登录验证码（无需鉴权）
```
POST /api/v1/auth/send-login-code
Body: { type: 'email' | 'phone', target: string }
Response: { code: 0, message: '验证码已发送' }
```
- 复用 `verify-code.service.ts` 的 `sendCode()` 方法，purpose='login'
- 无需authGuard（登录前无token）

#### 验证码登录（无需鉴权）
```
POST /api/v1/auth/login-by-code
Body: { type: 'email' | 'phone', target: string, code: string }
Response: { code: 0, data: { accessToken, refreshToken, user } }
```
- 验证验证码 → 查找用户 → 签发双token

### 2.2 实现方案

在 `user.routes.ts` 新增两个公开路由：
- `POST /auth/send-login-code` — 无需鉴权
- `POST /auth/login-by-code` — 无需鉴权

在 `user.controller.ts` 新增：
- `sendLoginCode()` — 调用 verify-code.service.sendCode()
- `loginByCode()` — 验证码校验 + 用户查找 + token签发

在 `validators/auth.ts` 新增：
- `sendLoginCodeSchema` — Zod校验
- `loginByCodeSchema` — Zod校验

## 3. 前端设计

### 3.1 登录页Tab切换

```
┌─────────────────────────────────────┐
│          MineChat (金属渐变)          │
│         ─── 信号线 ───              │
│                                     │
│  [账号密码]  [手机号]   [邮箱]       │  ← Tab切换
│  ─────────────────────              │
│                                     │
│  (根据Tab显示不同表单)               │
│                                     │
│  [ 登 录 ]                          │
│                                     │
│  还没有账号？注册                     │
└─────────────────────────────────────┘
```

### 3.2 Tab样式
- 金属金下划线指示当前Tab
- 切换时表单fade过渡

### 3.3 表单字段

| Tab | 字段1 | 字段2 | 按钮 |
|-----|-------|-------|------|
| 账号密码 | 账号(用户名/邮箱) | 密码 | 登录 |
| 手机号 | 手机号 | 验证码 + 发送按钮 | 登录 |
| 邮箱 | 邮箱 | 验证码 + 发送按钮 | 登录 |

### 3.4 高级感背景

z-index分层：
```
z:0   — 深空基底(#08090B) + 微光网格纹理
z:1   — ParticleBackground(含流星+星尘+连线+脉冲)
z:2   — 星系旋涡CSS层(多层叠加，增强深度)
z:10  — 磨砂玻璃登录卡片
```
