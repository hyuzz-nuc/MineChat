# 15-MineChat视觉体系V3设计规范文档

## 1. 设计语言定义

**核心设计语言：黑白灰金渐变金属质感**

摒弃青色(#00F5D4)等低饱和度儿童色彩，确立以黑/白/灰为基底、金属金为点睛的高级质感体系。灵感来源于高端腕表、奢华汽车仪表盘、深空探测器界面。

## 4. 宇宙动态背景设计

### 4.1 概述

开机动画结束后，登录页/聊天页背景不再是纯黑平面，而是注入宇宙动态元素：
- **流星划过**：随机角度的流星线条，带尾迹渐隐
- **星系旋涡**：中心微弱的螺旋光晕，缓慢旋转
- **星尘粒子**：继承现有ParticleEngine，但颜色改为暖金/银白

### 4.2 流星参数

| 参数 | 值 |
|------|-----|
| 数量 | 3-5条同时存在 |
| 长度 | 80-200px |
| 速度 | 300-600px/s |
| 颜色 | 头部`rgba(245,230,184,.8)` → 尾部`transparent` |
| 尾迹宽度 | 头部2px → 尾部0.5px |
| 出现间隔 | 2-5秒随机 |
| 角度 | 15°-45°（斜向下） |

### 4.3 星系旋涡参数

| 参数 | 值 |
|------|-----|
| 中心位置 | 屏幕中心偏上(50%, 40%) |
| 半径 | min(vw, vh) * 0.3 |
| 颜色 | `rgba(139,115,85,.04)` → `rgba(212,175,55,.02)` → `transparent` |
| 旋转速度 | 0.02 rad/s |
| 实现方式 | CSS radial-gradient + animation: rotate |

### 4.4 实现方案

**不新建引擎**，在现有 `particleEngine.ts` 中新增流星和旋涡：

```typescript
// particleEngine.ts 新增

interface Meteor {
  x: number; y: number;
  angle: number;     // 弧度
  speed: number;     // px/s
  length: number;    // px
  life: number;      // 剩余生命(0-1)
  maxLife: number;    // 总生命
  width: number;     // 头部宽度
}
```

流星用 Canvas 2D lineTo + 渐变线条绘制。
星系旋涡用 CSS 动画（更省性能），在 `LoginView.vue` 或 `App.vue` 中添加一层 div。

## 5. 交互闭环修复

### 5.1 问题分析

当前 SplashOverlay.vue 的 `dismiss()` 只是隐藏了 splash 层，但用户如果未登录，路由守卫会将 `/` 重定向到 `/login`。而 splash 覆盖在 router-view 之上，所以 splash 消失后用户看到的应该是 `/login` 页面。

**但如果用户已登录**，splash 消失后直接进入聊天页。

**实际Bug**：检查 `dismiss()` 函数 — 它设置了 `show.value = false`，但 router-view 在 splash 下面，应该可见。需确认是否是因为 splash 组件未从DOM移除（v-if vs v-show）或 z-index 问题。

### 5.2 修复方案

在 `SplashOverlay.vue` 的 `dismiss()` 中：
1. splash退出后 `show.value = false`，v-if 从DOM移除 ✅
2. 检查路由是否正确导航到 `/login`
